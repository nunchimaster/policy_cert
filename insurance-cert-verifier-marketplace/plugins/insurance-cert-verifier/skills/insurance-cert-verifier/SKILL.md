# 보험 증권 출력검증 (insurance-cert-verifier)

검증용 엑셀을 기준(정답)으로 삼아, 증권 PDF에 보장내용·보장금액이 올바로 출력됐는지
점검하고 결과를 엑셀에 기록한다.

## 핵심 전제 (먼저 이해할 것)

1. **증권 "PDF"는 두 가지 형식이 있다.** ① 페이지별 `.jpeg` + `manifest.json`을 담은
   **ZIP 아카이브**(pdfplumber/pdfminer로 안 열림), ② 스캔 이미지를 담은 **표준 PDF**(`%PDF-…`).
   `extract_pages.py`가 형식을 자동 감지해 ①은 ZIP으로 풀고, ②는 **PyMuPDF로 페이지를 이미지로
   렌더링**한다(텍스트 레이어 없는 스캔본도 처리). 이후 단계는 형식과 무관하게 같은 `pages.json`을 쓴다.
2. **tesseract OCR은 이 스캔에서 신뢰도가 낮다.** 한글은 오인식, 증권번호·금액 숫자도 가끔
   틀린다(예: 8025688294 → 8026688204). 따라서 **최종 값 판독은 반드시 클로드가
   `crop_region.py`로 만든 확대 이미지를 `view` 도구로 직접 보고** 한다. OCR(`index_certs.py`)은
   증권번호→페이지 1차 후보에만 쓰고, 경계는 footer를 눈으로 확정한다.
3. **매칭 키는 보종코드(숫자).** 증권 '계약사항(보험가입내용)' 표의 상품명 앞에 `(84445)`처럼
   보종코드가 인쇄돼 있다. 이걸로 가입금액을 찾고 "코드 존재 여부"를 판정한다.
   단, 하단 '가입상품 보장내용' 표의 **보장명칭에는 보종코드가 없다**(이름만 있음).
4. **계산식은 수식구분코드(상세탭 J열, SCRT_CACL_FRML_DCD)에 따라 다르다.** 연산자는 항상 곱셈.
   - **`01`(비율)**: `보장금액 = 가입금액 × 분자 ÷ 분모`. 예) 가입금액 1,000,000, 100/100 → 1,000,000;
     50/100 → 500,000. (대부분의 라인)
   - **`13`(절대금액)**: `보장금액 = 분자 ÷ 분모` (분모는 보통 1, **분자 자체가 원 단위 금액, 가입금액과 무관**).
     예) 50000/1 → 50,000 (MRI촬영 급여), 5000000/1 → 5,000,000 (혈전용해치료).
   - `verify_lib.calc_amount(가입금액, 분자, 분모, 수식구분코드)`가 이 분기를 처리한다.
   - **처음 보는 수식구분코드**가 나오면 자동계산하지 말고 `검토`로 두고 사유에 코드값을 적는다.
5. **텍스트 비교 규칙(확정):** 띄어쓰기·전각/반각·괄호/꺾쇠/따옴표/쉼표 등 **기호는 모두 무시**,
   그 외 한글/영문/숫자가 다르면 불일치. `verify_lib.normalize()`가 이 규칙을 구현한다.

## 입력

- 검증용 엑셀 `.xls` (시트: `대상요약` + 상세 시트 1개)
- 증권 PDF 파일들 (보통 프로젝트 폴더 `/mnt/project/` 에 있음; 업로드 폴더도 확인)

## 산출물

`<원본명>_검증.xlsx` (값 보존 복제 + 아래 갱신):
- **대상요약**: I열(확인내용)=종합판정, 끝에 `검증PDF/검증페이지/텍스트판정/금액판정/종합사유` 추가
- **검증상세(신규 시트)**: 라인별 1행 — `보종코드/보종명/증권번호/PDF파일/페이지/항목구분/엑셀값/증권출력값/결과/사유`
- 결과셀 색상: O=연두, X=연빨강, 검토=노랑

## 판정 표기

- **O** : 엑셀과 증권 출력이 일치
- **X** : 불일치. 사유에 **무엇이 어떻게 다른지** 구체적으로 (예: `보장금액 불일치: 기대 1,000,000 / 증권 500,000`)
- **검토** : 자동 판정 불가. 사유에 **왜 검토인지** (예: `증권 이미지 흐림으로 금액 판독 불가`, `증권에 보장명칭 블록 다수로 매칭 모호`)
- 종합 확인내용(I열): 모두 O면 `이상없음`; X 있으면 `X: <요약>`; 검토면 `검토: <요약>`;
  계약사항 표에 보종코드 없으면 `해당 증권에 보종코드 없음`

## 절차

환경 준비(최초 1회):
```bash
pip install xlrd openpyxl pillow pymupdf --break-system-packages -q
# 한글 OCR이 필요하면 (보조용):
mkdir -p /tmp/tessdata && cd /tmp/tessdata
curl -sL -o kor.traineddata https://raw.githubusercontent.com/tesseract-ocr/tessdata/main/kor.traineddata
cp /usr/share/tesseract-ocr/5/tessdata/eng.traineddata /tmp/tessdata/ 2>/dev/null
export TESSDATA_PREFIX=/tmp/tessdata
```
스크립트는 모두 `scripts/`에 있다. `scripts/`에서 실행하거나 경로를 맞춰 실행한다.

### 1단계 — 엑셀 파싱
```bash
python parse_excel.py --xls <원본.xls> --out parsed.json
```
보종코드별 `보장명칭/보장내역/금액라인(텍스트,분자,분모)` 과 `대상요약`(보종코드/증권번호/담당자)을 얻는다.
`증권번호`가 숫자가 아닌 행(예: `갱신계약`)은 증권이 없으므로 **검토 사유 '증권 없음'으로 건너뛴다**.

### 2단계 — 증권 페이지 추출 + 증권번호 인덱싱
PDF마다 (형식 자동 감지; 표준 PDF는 PyMuPDF로 렌더, 흐리면 `--zoom 3.5~4.0`):
```bash
python extract_pages.py --pdf <증권.pdf> --workdir /tmp/cert_<이름> [--zoom 3.0]
TESSDATA_PREFIX=/tmp/tessdata python index_certs.py \
  --pages /tmp/cert_<이름>/pages.json --out /tmp/cert_<이름>/index.json
```
`index.json`의 `groups`는 증권번호별 페이지 범위 **후보**다. 경계가 1~2장 어긋날 수 있으니,
의심되면 해당 footer를 크롭해 눈으로 확인한다:
```bash
python crop_region.py --image <경계.jpeg> --region footer --out /tmp/_f.png   # 그리고 view
```
모든 PDF의 결과를 합쳐 `증권번호 → (PDF파일, 시작페이지, 끝페이지)` 맵을 만든다.

### 3단계 — 보종코드별 검증 (클로드 직접 판독)
각 대상 보종코드에 대해:

**(a) 증권 위치 확정.** 증권번호로 PDF·페이지 범위를 찾는다. 없으면 → `해당 증권에 보종코드 없음`(검토), 기록 후 다음.

**(b) 계약사항 표에서 가입금액 + 코드 존재 확인.** 증권 첫 1~2장(계약사항/보험가입내용 표)을 크롭해 본다:
```bash
python crop_region.py --image <증권첫장.jpeg> --region contract --out /tmp/_c.png   # view
```
상품명이 `(보종코드)`로 시작하는 행을 찾아 **가입금액**을 읽는다.
표에 그 보종코드가 없으면 → `해당 증권에 보종코드 없음`(검토).

**(c) 보장내용 표에서 명칭·내역·보장금액 읽기.** 보장내용 표가 있는 장(들)을 크롭해 본다:
```bash
python crop_region.py --image <보장내용장.jpeg> --region coverage --out /tmp/_v.png   # view
```
엑셀 보장명칭(H 합본)과 **정규화 비교**해 해당 보장 블록을 찾는다. 한 보종코드의 엑셀 명칭이
`...><납입면제>`처럼 **여러 블록**을 합친 것일 수 있으니, 증권의 해당 블록들을 모두 모아 비교한다.
보장 블록의 **보장명칭/보장내역 텍스트**와 각 줄의 **보장금액**(예: 암수술/항암방사선/항암약물 1,000,000)을 읽는다.
보장내용이 다음 장으로 이어지면 이어지는 장도 크롭해 본다.

**(d) 점검① 텍스트.** `verify_lib.normalize()` 기준으로 엑셀 H↔증권 보장명칭, 엑셀 I↔증권 보장내역 비교.
일치 O / 다르면 X(무엇이 다른지) / 못 읽으면 검토.

**(e) 점검② 금액.** 금액라인마다 `verify_lib.calc_amount(가입금액, 분자, 분모, 수식구분코드)`로 기대값을 구해
증권 보장금액과 비교. (식01=가입금액×분자/분모, 식13=분자/분모 절대값) 한 줄이라도 다르면 그 줄을 X로,
사유에 `기대 X / 증권 Y` 적시. 처음 보는 수식구분코드면 검토 처리.

**(f) 결과 적재.** 아래 스키마로 한 보종코드 결과를 만든다. 페이지번호(계약사항/보장내용)와
증권에서 실제로 읽은 값을 반드시 함께 기록한다(신뢰성·추적성).

### 4단계 — 엑셀 기록
모든 결과를 `results.json`(리스트)로 모아:
```bash
python write_results.py --xls <원본.xls> --results results.json --out <원본명>_검증.xlsx
```
그 후 `present_files`로 결과 파일을 제시한다.

## 처리 규모 팁

- 증권당 보종코드가 수십 개다. 한 증권의 페이지(계약사항·보장내용)를 한 번에 크롭/판독해
  그 증권 소속 보종코드들을 **묶어서** 처리하면 효율적이다(증권번호 단위로 루프).
- 진행 중간에 `results.json`을 누적 저장하면 중단·재개에 안전하다.
- 검증 불가 행(갱신계약 등)도 빠뜨리지 말고 검토 사유와 함께 기록한다.

## 예시 판정

- 입력: 84445, 식01, 가입금액 1,000,000, 100/100, 증권 보장금액 1,000,000 →
  출력: 금액 `O` (1,000,000 = 1,000,000×100/100)
- 입력: 84479, 식01, 가입금액 1,000,000, 1/100, 증권 10,000 → 출력: `O` (1,000,000×1/100=10,000)
- 입력: 84457, **식13**, 가입금액 20,000,000(무관), 50000/1, 증권 50,000 →
  출력: 금액 `O` (식13이므로 분자 50,000이 곧 보장금액. 가입금액 곱하지 않음)
- 입력: 식01, 분자/분모 50/100, 가입금액 1,000,000, 증권 보장금액 1,000,000 →
  출력: 금액 `X`, 사유 `기대 500,000 / 증권 1,000,000`
- 입력: 증권 계약사항 표에 해당 보종코드 행 없음 →
  출력: 확인내용 `해당 증권에 보종코드 없음`, 텍스트/금액 `검토`
